1) 공지 수집
학교 홈페이지 / 학과 홈페이지에서 공지사항 크롤링
공지 제목, 본문, 카테고리, 원문 링크, 게시일 수집
중복 공지 여부 확인
DB에 저장
2) 공지 저장
notices 컬렉션(또는 테이블)에 저장
같은 공지가 중복 저장되지 않도록 source_url 또는 별도 해시값 기준으로 중복 체크
최신 공지 우선 정렬 가능하도록 날짜 정보 저장
3) 공지 목록 조회
사용자는 메인 공지 페이지 진입
최신 공지 리스트 확인
검색어 입력 가능
카테고리/출처별 필터 가능
4) 공지 상세 조회
공지 리스트에서 항목 클릭
상세 화면에서 제목, 날짜, 카테고리, 본문, 원문 링크 확인
5) 커뮤니티 연결
공지와 관련된 질문/의견은 게시판에서 소통
공지와 연결된 질문글 작성 기능 제공

GET /api/notices 공지 목록 조회
GET /api/notices/:noticeId 공지 상세 조회
POST /api/internal/notices/crawl 공지 크롤링 동기화

POST /api/posts 게시글 작성
GET /api/posts 게시글 목록 조회
GET /api/posts/:postId 게시글 상세 조회
POST /api/posts/:postId/likes 좋아요 등록
DELETE /api/posts/:postId/likes 좋아요 삭제
PATCH /api/posts/:postId 게시글 수정
DELETE /api/posts/:postId 게시글 삭제

POST /api/posts/:postId/comments 댓글 작성
GET /api/posts/:postId/comments 댓글 조회
PATCH  /api/posts/:postId/comments/:commentId 댓글 수정
DELETE /api/posts/:postId/comments/:commentId 댓글 삭제
